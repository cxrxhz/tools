# 概率决定系统

本项目是一个基于浏览器的事件许可与奖励管理系统，适用于自控、奖励、决策等场景。用户可自定义事件、设置概率与奖励规则，通过概率和奖励值系统辅助决策。

## 功能简介

- **事件许可决策**：根据时间间隔、迫切程度和奖励值，动态计算事件许可概率。
- **奖励值系统**：奖励值可用于提升许可概率，也可作为请求许可的消耗。
- **任务系统**：支持周期性任务，完成任务可获得奖励值。
- **事件自定义**：可自定义事件名称、概率参数、权重、奖励消耗等。
- **数据导入导出**：支持配置的导入导出，便于备份和迁移。

## 使用说明

1. 打开 `概率决定.html`，推荐使用现代浏览器（如 Chrome、Edge）。
2. 在“许可页面”选择事件和迫切程度，点击“请求许可”进行决策。
3. 可输入奖励值提升概率，或设置事件的“提问消耗”作为请求许可的门槛。
4. 在“编辑事件”页可自定义事件参数，包括概率、权重、奖励消耗等。
5. 通过“编辑任务”可添加周期性任务，完成后获得奖励值。
6. 可导出/导入全部配置数据。

## 概率与奖励机制

- 概率由时间间隔、迫切程度、奖励值共同决定。
- 迫切程度分为1~8级，对应概率值和动态权重，权重放大规则如下：
  - 1~4级：权重分别为8/6/4/2倍，概率更低
  - 5级：权重1倍
  - 6~8级：权重分别为2/4/6倍，概率更高
- 奖励值可直接提升概率，提升幅度递减。
- 事件可设置“提问消耗”，无足够奖励值无法请求许可。

## 主要界面

- **许可页面**：事件选择、迫切程度、奖励值输入、请求许可、倒计时、结果显示。
- **编辑事件**：事件参数自定义，包括概率、权重、奖励消耗等。
- **任务系统**：周期任务管理，完成任务获得奖励值。
- **导入导出**：配置数据的备份与恢复。

## 技术说明

- 纯前端实现，无需后端，数据存储于浏览器 localStorage。
- 主要技术：HTML、JavaScript、CSS。

## 服务器模式配置详细说明

本系统支持“本地模式”与“服务器模式”两种配置存储方式，方便多设备同步和团队协作。

### 一、服务器端部署与运行

1. **准备环境**

   - 需安装 Python 3。
   - 推荐安装 Flask 库：
     ```bash
     pip install flask
     ```
2. **准备文件**

   - 将 `概率决定.html` 和下方示例 `event_server.py` 放在同一目录。
3. **示例服务器代码（event_server.py）**

   ```python
   from flask import Flask, request, jsonify, send_from_directory
   import json, os
   app = Flask(__name__)
   CONFIG_PATH = 'event_config.json'

   @app.route('/')
   def index():
       return send_from_directory('.', '概率决定.html')

   @app.route('/config', methods=['GET', 'POST'])
   def config():
       if request.method == 'POST':
           with open(CONFIG_PATH, 'w', encoding='utf-8') as f:
               f.write(request.data.decode('utf-8'))
           return jsonify({'status': 'ok'})
       if os.path.exists(CONFIG_PATH):
           with open(CONFIG_PATH, 'r', encoding='utf-8') as f:
               return f.read(), 200, {'Content-Type': 'application/json'}
       return jsonify({'events': [], 'rewardStock': 0, 'tasks': []})

   if __name__ == '__main__':
       app.run(host='0.0.0.0', port=5000)
   ```
4. **运行服务器**

   - 在命令行进入上述目录，运行：
     ```bash
     python event_server.py
     ```
   - 若提示 Flask 未安装，请先执行 `pip install flask`。
5. **访问方式**

   - 在服务器本机浏览器访问：

     ```
     http://localhost:5000/
     ```
   - 在同一局域网其他设备（如手机、电脑）访问：

     ```
     http://<服务器IP>:5000/
     ```

     其中 `<服务器IP>` 可通过 `ipconfig`（Windows）或 `ifconfig`（Linux/Mac）查询。

### 二、服务器模式使用说明

- 顶部右侧“本地模式/服务器模式”按钮可切换配置来源。
- 切换到“服务器模式”时，系统会自动将本地配置上传到服务器，并从服务器拉取最新配置应用到当前页面。
- 切回“本地模式”时，恢复为本地浏览器存储的配置。
- 切换失败（如服务器不可用）会自动回退到本地模式并提示。

### 三、配置同步机制

- 切换到服务器模式时，所有配置（事件、奖励值、任务等）会自动同步到服务器。
- 在服务器模式下，所有操作（保存、导入、编辑）都会实时同步到服务器。
- 本地模式下，所有数据仅存储于当前浏览器，不与其他设备同步。

### 四、常见问题与建议

- 服务器模式需保证浏览器与服务器在同一网络，且允许跨设备访问。
- 若服务器不可用或网络异常，系统会自动回退到本地模式，避免数据丢失。
- 推荐定期导出配置备份。
- 如需公网访问，请配置端口映射或使用云服务器。
